## 三、代码规范

### 核心原则
1. **最小改动**：只改需要改的地方，禁止重构
2. **保护正确代码**：已验证正确的部分绝对不动
3. **透明修改**：每次修改必须说明原因和影响
4. **先定位后修改**：优先定位问题，不直接改代码

### Excel处理规范
- **禁止覆盖**：H/J列公式列禁止代码覆盖
- **E-K列保护**：无明确要求不改动
- **动态跳过**：小计/合计/汇总行自动跳过

### 详细规范
见 `references/coding-standards.md`

## 五、算法计算口径

### 目标L值计算
```
目标L值 = (Z-1)/9*25 + (AA-1)/9*25 + (AB-1)/9*50
```
- 任一值 < 1 时，该项贡献为0
- 结果保留4位小数

### 列填充规则（业务降本推进）
| 列 | 条件 | 计算 |
|----|------|------|
| E | L>=1 且 T长度>=10 | 计数 |
| F | L>=1 | 计数 |
| G | L>=1 | 求和 |
| H | L>=1 且 X="是" | 计数（公式） |
| I | X="是" | 计数（无L限制）|
| K | L>=1 且 Z>=1 | 平均值 |
| L | - | K列分子 |
| M | - | K列分母 |

### 详细算法
见 `references/algorithm-library.md`

## 六、踩坑记录

### 已解决的坑
1. **Excel样式复制**：使用临时文件法解决
2. **公式列覆盖**：动态检测合并单元格行并跳过
3. **大文件内存**：read_only模式+及时关闭
4. **线程安全**：Tkinter用after()，Qt用Signal
5. **日期格式**：统一解析多种格式
6. **配置文件损坏**：备份+默认配置

### 详细踩坑记录
